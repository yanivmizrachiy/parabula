# rules.md — כללי הפרויקט (חובה)

## מטרת הפרויקט

אתר “ספר לימוד” בעברית, דפי A4 עקביים, ניווט נוח, הדפסה מושלמת, ועדכון אוטומטי מ־GitHub.

## עקרונות קבועים

- דפי A4 בלבד (מסך + הדפסה).
- ניווט בעברית בכל עמוד: עמוד קודם / עמוד הבא / תצוגת הדפסה / הדפס.
- אין קבצים מיותרים בריפו.
- כל שינוי מבני/עיצובי/תפעולי נרשם כאן לפני פרסום.

## היסטוריית שינויים

- 2026-03-03: הוקם מנגנון אתר ספר אוטומטי (site), ניווט, הדפסה, ופרסום ב־Pages.
- 2026-03-03: Automation scripts updated (build-site.ps1, verify-all.ps1, publish.ps1).
- 2026-03-03: Automation verification and report updated (verify-all.ps1, REPORT.md).

## מה נותר לעשות

- להוסיף נושאים נוספים באותו מבנה.
- לשפר גרפיקת “פרימיום” לפי שפה עיצובית שנבחר.

## אתר חי ואייקון קבוע

- האתר מתפרסם אוטומטית דרך GitHub Pages מתוך `/site`.
- כל `git push` מסנכרן את התצוגה החיצונית.
- קיים קיצור דרך על שולחן העבודה בשם **Parabula – ספר לימוד** שמוביל לקישור החי.
- אין לפתוח קבצי HTML מקומיים לצפייה — רק דרך הקישור החי.

## תצוגה מקדימה (/preview)

- ב-`/preview` כפתורי הניווט (מצב תצוגה, הקודם/הבא, וכפתורי נושאים) חייבים להישאר גלויים גם בזמן גלילה.

## הפרדה מלאה בין HTML ל-CSS (חובה בכל הפרויקט)

- בכל קבצי ה-HTML בפרויקט **אסור** להשתמש ב-CSS בתוך ה-HTML:
	- אין `<style>...</style>`
	- אין `style="..."`
- כל עיצוב חייב להיות בקבצי CSS נפרדים בלבד.
- בדפי A4 (עמוד-N.html):
	- CSS בסיס: `styles/a4-base.css`
	- CSS ייעודי לעמוד: `styles/pages/עמוד-N.css`
- בדפי נושאים (pages/* שמייצרים את site/*):
	- CSS של הנושא חייב להיות בקובץ `style.css` של הנושא (או בקובץ ה-CSS של העמוד, אם קיים), ולא בתוך ה-HTML.

## כתיב מתמטי

- הכתיב המתמטי בספר אחיד ומבוסס על MathJax 3.
- כל נוסחה נכתבת ב-LaTeX בלבד.
- בתוך שורה: `\( ... \)`
- נוסחה מוצגת: `

\[ ... \]

`

- אסור להשתמש ב-HTML ידני (sup/sub) לכתיב מתמטי.
- חובה לבדוק מסך + הדפסה (A4) לפני פרסום.
