מדיניות: לא מוסיפים URL-ים חיצוניים נוספים בלי הוראה מפורשת. אם עובדים אופליין—ייתכן שהפונט/MathJax לא ייטענו; הדפים עדיין צריכים להישאר תקינים מבנית.

מבנה בסיס חובה לכל דף A4:
התאמת נושא (דרישה): התוכן של כל עמוד חייב להתאים לנושא שמופיע ב-<title> וב-.preview-nav .nav-meta. לא מעתיקים שאלות/תרגילים של נושא אחד לעמוד ששייך לנושא אחר.
<main class="a4-page page-1"> <header class="header-container"> <h1 class="page-title">כותרת</h1> <div class="page-number">1</div> </header> <div class="question-block">...</div> </main>
כותרת הדף תמיד ב-.page-title, ומספר עמוד תמיד בעיגול .page-number.
כותרות: הכותרות בכל הפרויקט (כלומר .page-title) הן כתב כחול (var(--title-blue)) ללא הדגשה (משקל רגיל; לא bold).
לא מוסיפים מספור ידני לסעיפי משנה בתוך הטקסט (א./ב./ג. וכו') כאשר כבר יש בולט. סעיפי משנה נכתבים כ-.q-sub וההפרדה ביניהם נעשית באמצעות הבולט בלבד.
כלל מספור: ברירת המחדל למספור פנימי היא מספרים 1, 2, 3… (למשל בטבלה/רשימת פריטים/פתרונות). אותיות עבריות (א, ב, ג…) מותרות רק כאשר זה חלק מדרישת ה-UX של הדף (למשל סימון תרגילים א–ו בדפי תרגול), ובמקרה כזה זה מוגדר בעמוד עצמו.
כלל תוכן חובה: השאלות עצמן לא ממוספרות — לא כותבים שאלה 1, שאלה 2 וכו'. כל שאלה מתחילה בבולט שחור (לפי רכיבי הבסיס), ורק העמודים ממוספרים בעיגול .page-number בהתאם לעיצוב הרגיל של הפרויקט.
כל דף עבודה חייב “רקע/עיצוב ייעודי” לפי שם העמוד (עמוד 1, עמוד 2 וכו').
היישום התקני: לכל עמוד קיימת מחלקה ייחודית על ה-.a4-page (למשל page-1, page-2), ובנוסף קובץ CSS ייעודי לעמוד שמכיל רק overrides/רקע של אותו עמוד.
<link rel="stylesheet" href="styles/a4-base.css"> <link rel="stylesheet" href="styles/pages/עמוד-1.css">
קובץ הבסיס נשאר אחיד לכל הפרויקט; קובץ עמוד ייעודי לא משנה טוקנים/פונטים—רק התאמות נקודתיות לאותו עמוד.
הדפסה A4:
חובה לשמור על @page { size: A4; margin: 0; } ועל ביטול הצללה/שוליים למסך בתוך @media print, כפי שמוגדר ב-styles/a4-base.css.
אסור לשנות את מידות .a4-page (210mm × 297mm) ואת ה-padding שלו בלי הוראה מפורשת ממך.
תצוגה מקדימה (דפדוף כמו ספר):
כל עמוד HTML כולל פס ניווט .preview-nav עם כפתורי “הקודם/הבא” כדי לדפדף בנוחות בין דפים. הפס מוצג רק במסך ומוסתר בהדפסה בתוך @media print.
בנוסף יש Reader תצוגה מקדימה בכתובת /preview (דרך השרת npm run preview:node או ./preview.ps1). ברירת המחדל היא פורט 5179, לדוגמה: http://127.0.0.1:5179/preview. ה-Reader מציג עמוד אחד בכל רגע, עם כפתורי דפדוף נוחים.
אפשר לפתוח ישירות קובץ מסוים בתוך ה-Reader באמצעות פרמטר file, לדוגמה: /preview?file=עמוד-3.html.
פיתוח מהיר (Vite): npm run dev מרים שרת מקומי על http://localhost:5173/ (או פורט אחר אם 5173 תפוס) ומאפשר לפתוח קבצי עמוד ישירות (למשל /עמוד-9.html).
ה-Reader כולל שני מצבי תצוגה: "כל הדפים" — תצוגה רציפה של כל הדפים לפי סדר הקריאה (גלילה כמו ספר לימוד), ו-"דפדוף" — תצוגת ספר (עמוד/זוג עמודים) עם כפתורי “הקודם/הבא”.
כלל ברזל: ב-/preview חייבת להיות תצוגה קבועה של כל דפי העבודה ברצף (מצב “כל הדפים”), כך שאפשר לקרוא/לסקור את כל החומר בסקרול אחד בלי דפדוף ידני בין קבצים. מצב “כל הדפים” הוא ברירת המחדל.
מספור עמודים מתאפס בכל נושא חדש: כאשר מתחיל נושא חדש (כותרת חדשה), הערך בתוך .page-number חוזר ל-1, וגם הטקסט ב-.preview-nav .nav-meta מציג “שם הנושא — עמוד X / Y”.
פס הניווט כולל גם אזור “מעבר בין נושאים” (.preview-nav-topics) עם כפתורי נושא מעוצבים (.topic-link). כפתורים אלו מוצגים רק בתצוגה (מסך) ומוסתרים בהדפסה.
דרישה חובה: גם בתוך ה-Reader בכתובת /preview חייב להיות אזור “מעבר בין נושאים” עם כפתורי נושא (לחצנים). הכפתורים חייבים להיות זמינים תמיד — גם במצב "כל הדפים" וגם במצב "דפדוף" — כדי לאפשר מעבר מיידי לנושא אחר בלי לחפש בעמודים.
דרישה חובה: בטעינה ראשונית של /preview לא מתחילים במסך ריק. מוצג טקסט מצב “טוען…” עד שהעמוד הראשון נטען ומוצג בפועל.
דרישה: במצב "כל הדפים" חייב להיות גובה התחלתי לתצוגת העמודים (iframe) כך שניתן לראות עמודים גם לפני שה-fit/zoom רץ, ולא מתקבלת תצוגה "לבנה".
כאשר אין מקום בעמוד: מפצלים לעמוד הבא בקובץ חדש (למשל עמוד-5.html) עם קובץ CSS ייעודי (למשל styles/pages/עמוד-5.css), ומעדכנים את כל קישורי הניווט הקודם/הבא, ואת הטקסט עמוד X / Y בכל עמודי הנושא.
דוגמה בפועל: אם הוסף “עמוד המשך” באמצע נושא (למשל עמוד-6.html), חייבים לעדכן את מספור העמוד בעיגול (.page-number) ואת עמוד X / Y בכל העמודים הרלוונטיים של אותו נושא.
RTL / LTR:
המסמך כולו dir="rtl" (עברית). רכיבים מתמטיים/טבלאות/מערכות צירים משתמשים ב-direction: ltr לפי המחלקות הקיימות.
מערכת צירים (תוויות וריווח):
תוויות הצירים x/y מוגדרות בבסיס (styles/a4-base.css). האות y חייבת להיות בדיוק מעל החץ של ציר ה-y ולכן היא ממורכזת ע"י left: 50% יחד עם transform: translateX(-50%).
כלל חובה (דרישה חזקה): הצירים עצמם (הקווים השחורים) חייבים לחפוף לקווי הרשת: גם ציר x וגם ציר y — ובפרט ציר ה-y חייב לעבור בדיוק על קו רשת (לא בין שתי משבצות). כדי לקבל דיוק פיקסל-לפיקסל, עובי הצירים הוא 1px והם ממוקמים בדיוק ב-50% (ללא transform). בנוסף, גודל .coordinate-system בעמודים ייעודיים חייב להיות מכפלה של 44px (כלומר מספר זוגי של משבצות בגודל 22px) כדי שהאמצע יהיה בדיוק קו רשת. אסור להזיז את הרשת (למשל עם background-position) כדי “לפגוש” את הציר; מתקנים יישור אך ורק באמצעות מיקום הצירים/הגודל, כך שהציר יישב על קו רשת.
/* styles/a4-base.css */ .label-y { left: 50%; top: -25px; transform: translateX(-50%); }
אם מערכת הצירים צמודה מדי לטקסט שמעליה—לא משנים padding כללי של הדף. במקום זה מוסיפים ריווח נקודתי בקובץ ה-CSS הייעודי של אותו עמוד (למשל margin-top ל-.coordinate-container).
MathJax:
נוסחאות inline נכתבות תמיד בפורמט \( ... \). אין להכניס רנדר ידני או ספריות חלופיות בלי עדכון לכללי המסמך.
SVG (תוויות וריווח):
דרישה חובה: בכל שרטוטי ה-SVG, כל אות/מספר/סימן שליד הציור (כולל A/B/C, אורכי צלעות, ו-x) נכתב כ-MathJax באמצעות \( ... \) בתוך אלמנט HTML (Overlay), לא כ-<text> בתוך ה-SVG.
מיקום התוויות חייב להיות סימטרי ומותאם לגודל הצלע: ממורכז לאורך הצלע הרלוונטית, במרחק “נכון” מהקו (לא צמוד מדי ולא רחוק מדי), כדי להיראות כמו דף עבודה מודפס.
אזור הכתיבה לפתרון תלמידים משתמש ברשת 22px (background-size: 22px 22px) וחייב להכיל מספיק שורות; כשמתבקשת “עוד שורת משבצות”, מגדילים את הגובה הפנימי בפקטור של 22px.
פיתגורס (דרישה): כאשר נדרש “אותו עיצוב כמו עמוד 1 של פיתגורס”, בונים את העמוד כגריד של כרטיסי תרגיל (figure + .solution-space) עם פוטר תשובות — ולא כתמונה חתוכה של PDF.
איסורים לשמירה על אחידות:
