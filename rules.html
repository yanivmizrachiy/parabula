R1
מטרת המסמך: להגדיר סטנדרט אחיד ליצירת דפי עבודה להדפסה A4, באותו סגנון בדיוק, עם הפרדה מוחלטת בין HTML ל-CSS.
כל שינוי בסגנון/כללים מתבצע אך ורק ע"י עריכה למסמך זה, ובהתאם להנחיות שתמסור.
כלל עליון: אין CSS בתוך ה-HTML.
אסור להשתמש ב-<style>, ואסור להשתמש במאפיין Inline Style על תגיות. כל עיצוב—רק בקבצי CSS.
HTML מכיל רק מבנה/טקסט/MathJax/מחלקות. CSS מכיל את כל ה-layout והעיצוב.
מבנה בסיס חובה לכל דף A4:
<main class="a4-page page-1"> <header class="header-container"> <h1 class="page-title">כותרת</h1> <div class="page-number">1</div> </header> <div class="question-block">...</div> </main>
כותרת הדף תמיד ב-.page-title, ומספר עמוד תמיד בעיגול .page-number.
לא מוסיפים מספור ידני לסעיפי משנה בתוך הטקסט (א./ב./ג. וכו') כאשר כבר יש בולט. סעיפי משנה נכתבים כ-.q-sub וההפרדה ביניהם נעשית באמצעות הבולט בלבד.
כל דף עבודה חייב “רקע/עיצוב ייעודי” לפי שם העמוד (עמוד 1, עמוד 2 וכו').
היישום התקני: לכל עמוד קיימת מחלקה ייחודית על ה-.a4-page (למשל page-1, page-2), ובנוסף קובץ CSS ייעודי לעמוד שמכיל רק overrides/רקע של אותו עמוד.
<link rel="stylesheet" href="styles/a4-base.css"> <link rel="stylesheet" href="styles/pages/עמוד-1.css">
קובץ הבסיס נשאר אחיד לכל הפרויקט; קובץ עמוד ייעודי לא משנה טוקנים/פונטים—רק התאמות נקודתיות לאותו עמוד.
הדפסה A4:
חובה לשמור על @page { size: A4; margin: 0; } ועל ביטול הצללה/שוליים למסך בתוך @media print, כפי שמוגדר ב-styles/a4-base.css.
אסור לשנות את מידות .a4-page (210mm × 297mm) ואת ה-padding שלו בלי הוראה מפורשת ממך.
תצוגה מקדימה (דפדוף כמו ספר):
כל עמוד HTML כולל פס ניווט .preview-nav עם כפתורי “הקודם/הבא” כדי לדפדף בנוחות בין דפים. הפס מוצג רק במסך ומוסתר בהדפסה בתוך @media print.
בנוסף יש Reader תצוגה מקדימה בכתובת /preview (דרך השרת npm run preview). ה-Reader מציג עמוד אחד בכל רגע, עם כפתורי דפדוף נוחים.
מספור עמודים מתאפס בכל נושא חדש: כאשר מתחיל נושא חדש (כותרת חדשה), הערך בתוך .page-number חוזר ל-1, וגם הטקסט ב-.preview-nav .nav-meta מציג “שם הנושא — עמוד X / Y”.
פס הניווט כולל גם אזור “מעבר בין נושאים” (.preview-nav-topics) עם כפתורי נושא מעוצבים (.topic-link). כפתורים אלו מוצגים רק בתצוגה (מסך) ומוסתרים בהדפסה.
כאשר אין מקום בעמוד: מפצלים לעמוד הבא בקובץ חדש (למשל עמוד-5.html) עם קובץ CSS ייעודי (למשל styles/pages/עמוד-5.css), ומעדכנים את כל קישורי הניווט הקודם/הבא, ואת הטקסט עמוד X / Y בכל עמודי הנושא.
דוגמה בפועל: אם הוסף “עמוד המשך” באמצע נושא (למשל עמוד-6.html), חייבים לעדכן את מספור העמוד בעיגול (.page-number) ואת עמוד X / Y בכל העמודים הרלוונטיים של אותו נושא.
RTL / LTR:
המסמך כולו dir="rtl" (עברית). רכיבים מתמטיים/טבלאות/מערכות צירים משתמשים ב-direction: ltr לפי המחלקות הקיימות.
מערכת צירים (תוויות וריווח):
תוויות הצירים x/y מוגדרות בבסיס (styles/a4-base.css). האות y חייבת להיות בדיוק מעל החץ של ציר ה-y ולכן היא ממורכזת ע"י left: 50% יחד עם transform: translateX(-50%).
דרישה חזקה: צירי x/y חייבים להיות בדיוק על קווי הרשת (לחפוף לקו רשת). הצירים חייבים להיות ממורכזים על האמצע (כלומר ממורכזים סביב 50% בעזרת transform), ולא “להתחיל” ב-50%. בנוסף, גודל .coordinate-system בעמודים ייעודיים חייב להיות מכפלה של 44px (כלומר מספר זוגי של משבצות בגודל 22px), כדי שהקו האמצעי יהיה קו רשת. אסור להזיז את הרשת עם background-position באופן שגורם לצירים “ליפול בין משבצות”.
/* styles/a4-base.css */ .label-y { left: 50%; top: -25px; transform: translateX(-50%); }
אם מערכת הצירים צמודה מדי לטקסט שמעליה—לא משנים padding כללי של הדף. במקום זה מוסיפים ריווח נקודתי בקובץ ה-CSS הייעודי של אותו עמוד (למשל margin-top ל-.coordinate-container).
MathJax:
נוסחאות inline נכתבות תמיד בפורמט \( ... \). אין להכניס רנדר ידני או ספריות חלופיות בלי עדכון לכללי המסמך.
איסורים לשמירה על אחידות:
לא מוסיפים צבעים/פונטים/צללים חדשים מעבר לטוקנים שמוגדרים ב-:root של קובץ הבסיס—אלא אם תבקש במפורש.
לא מוסיפים הדגשות למלל מיוזמתנו (בולד/קו תחתון/תיבות “הדגשה”/שינוי משקל פונט) — מדגישים רק מילים שאתה אומר במפורש להדגיש.
לא מוסיפים “דמו”. כל עמוד עבודה שניצור בהמשך יכיל רק את התוכן שאתה מגדיר, והסגנון נגזר מהכללים כאן.
